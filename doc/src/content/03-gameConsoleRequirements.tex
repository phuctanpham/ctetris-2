\section{Đặc tả Yêu cầu Nghiệp vụ (SRS): Module gameConsole}

\subsection{Tổng quan Nghiệp vụ (Business Overview)}
Phân hệ \textbf{gameConsole} đóng vai trò là Trung tâm điều phối (Hub Screen) của ứng dụng \textit{cTetris}. Module này quản lý trạng thái của hệ thống ngoài luồng chơi chính, cung cấp giao diện để người dùng khởi động ván đấu, tra cứu hướng dẫn, xem bảng xếp hạng, thiết lập cấu hình cá nhân hóa và quản lý việc thoát ứng dụng an toàn trên đa nền tảng.

\subsection{Mô hình hóa Hệ thống (System Modeling)}

\subsubsection{Kiến trúc Cơ sở dữ liệu (SQLite --- 3 nhóm bảng)}
\textit{ERD đầy đủ với DDL-level precision (8 bảng, tên thực
\texttt{default\_Records}/\allowbreak\texttt{default\_Stories}/\allowbreak\texttt{default\_Settings},
foreign-key relationships, 3 ownership groups) được đặc tả trong
\textbf{SDS §06-gameConsoleEngineering, mục 2.4}.}

SRS chỉ phát biểu ràng buộc schema:
\begin{itemize}[nosep]
  \item File: \texttt{SDL\_GetPrefPath("uit","cTetris")} $+$
      \texttt{idUser} $+$ \texttt{".sqlite"}.
  \item User-prefix tables: \texttt{g\_dbCurrentUser + "\_" + suffix}
      (Group 1 --- R/W owner).
  \item \texttt{shared\_*}: schema owner = \texttt{gameConsole}; gameStory
      populates content (Group 2 --- read-only to gameConsole).
  \item \texttt{sync\_Records}: D1-fed leaderboard sink; seeded fallback rows
      (Group 3).
  \item \texttt{dbInitSchema()} idempotent --- \texttt{CREATE TABLE IF NOT EXISTS}
      for all 8 tables; safely re-runnable on each entry.
\end{itemize}

\subsubsection{Hợp đồng dữ liệu \texttt{SettingsConfig} (Cross-module Contract)}
\textit{Component diagram và sequence diagram của \texttt{SettingsConfig}
flow (Console $\leftrightarrow$ Core $\leftrightarrow$ DBLayer) được đặc tả trong
\textbf{SDS §06-gameConsoleEngineering, mục 2.5}.}

SRS phát biểu ràng buộc contract:
\begin{itemize}[nosep]
  \item \textbf{Persisted fields} (10 rows in \texttt{default\_Settings}):
      \texttt{volume}, \texttt{color0-6}, \texttt{storyId}, \texttt{chapterId}.
  \item \textbf{Runtime-only fields} (NOT persisted):
      \texttt{nextBlockScore}, \texttt{nextBlockSpeed}, \texttt{tableMatrix} ---
      set from Stories popup, forwarded to \texttt{runGameCore()}.
  \item Single \texttt{volume} field (no separate bgm/sfx split in V2).
  \item \texttt{SettingsConfig\&} is a \textbf{borrowed pointer} inside
      \texttt{AppState::cfg} --- \texttt{runGameConsole()} does not own it.
\end{itemize}

\subsection{Yêu cầu Phiên bản V1: Nền tảng Hub Screen \& Lightbox UI}
\begin{itemize}[leftmargin=*]
    \item \textbf{Mục tiêu cốt lõi:} Xây dựng bộ khung UI tiêu chuẩn, hệ thống điều hướng đa kênh (chuột, phím, con lăn) và quản lý Pop-up (Lightbox) bằng mảng dữ liệu tĩnh.
    \item \textbf{Yêu cầu Chức năng:}
    \begin{itemize}[noitemsep]
        \item Hệ thống Pop-up Lightbox: Phủ mờ nền 70\%, tích hợp Scrollbar hỗ trợ kéo thả và tự động cuộn.
        \item Điều hướng đa năng: Hỗ trợ WASD, Mũi tên, TAB cho việc chuyển đổi Focus; Enter/Space để chọn.
        \item Xử lý văn bản: Thuật toán Word Wrapping tự động ngắt dòng tại 28 ký tự để đảm bảo hiển thị đẹp trong popup.
    \end{itemize}
    \item \textbf{Yêu cầu Phi Chức năng:} Đảm bảo thời gian phản hồi phím < 50ms. Tích hợp ký hiệu Copyright (©) UTF-8 trên thanh tiêu đề OS.
    \item \textbf{Yêu cầu Kỹ thuật \& Tiêu chí thành công:}
    \begin{itemize}[noitemsep]
        \item Kỹ thuật: Sử dụng cấu trúc \texttt{gameConsole\_layout.h}. Chế độ \texttt{BUILD\_STANDALONE} cho kiểm thử độc lập.
        \item \textbf{Tiêu chí thành công:} Chuyển đổi trạng thái Focus mượt mà; Popup không bị lỗi tràn văn bản; Scrollbar hoạt động chính xác theo tọa độ chuột.
    \end{itemize}
\end{itemize}

    \item \textbf{Background SVG (2.1):} Lazy-rasterize 1 lần/session qua nanosvg $\rightarrow$ \texttt{SDL\_Texture}. Reset pointer khi exit để tránh leak khi re-entry. WASM: embed SVG dạng header C string, không cần \texttt{--preload-file}.
    \item \textbf{Popup Settings (2.3):} Volume slider drag (\texttt{SDL\_AudioStream} + \texttt{SDL\_SetAudioStreamGain}); phím LEFT/RIGHT ±5\%; 7 color swatches toggle multi-select với guard tối thiểu 1 màu. Đóng popup tự gọi \texttt{dbSaveSettings()}.
    \item \textbf{Popup Stories (2.6):} 3 trạng thái hàng — \textbf{Locked} / \textbf{Active} / \textbf{Completed}; navigator chapter trái/phải; nút Play chuyển scene về gameStory preview (return 2). Click radio chọn story $\rightarrow$ \texttt{dbSelectStory()} + cập nhật \texttt{SettingsConfig}.
    \item \textbf{Leaderboard từ DB (2.7):} \texttt{sync\_Records} ORDER BY totalScore DESC; fallback \texttt{FALLBACK\_BOARD\_ROWS[]} khi rỗng; sort 4 chế độ qua \textit{Smart Sort Engine} (Insertion cho $n \leq 64$, Intro cho $n > 64$).
    \item \textbf{Unlock cascade (2.6):} \texttt{dbCheckAndUnlockStories()} mỗi lần vào Console: parse CSV \texttt{requiredStories}, so sánh với \texttt{lastMaxScore/Speed/Retries}, UPSERT \texttt{isActivated=1} nếu đủ điều kiện.
\end{itemize}

\subsubsection{Yêu cầu Phi Chức năng V2}
\begin{itemize}[noitemsep, leftmargin=*]
    \item SQLite compile flag: \texttt{-DSQLITE\_THREADSAFE=0} (WASM single-thread).
    \item \texttt{dbInitSchema()} idempotent — \texttt{CREATE TABLE IF NOT EXISTS} toàn bộ 9 bảng.
    \item Mọi write có \texttt{dbSyncToPersistent()} ngay sau để đảm bảo IDBFS flush trên WASM.
    \item \texttt{SettingsConfig} thay đổi struct bắt buộc cập nhật cả \texttt{runGameCore()} signature.
\end{itemize}
